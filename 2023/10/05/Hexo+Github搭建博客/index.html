Hexo+Github搭建博客

发表于2023-10-05|更新于2023-10-05
|字数总计:2.2k|阅读时长:7分钟|阅读量:

Hexo准备

下载安装Git

Git官网

下载电脑对应的版本（科学上网下载会比较快一点）

一直点next，默认安装就可以了。

出现“Open Git GUI here” 和“Open Git Bash here”这两个选项就代表安装成功了

“Open Git Bash here”是我们经常用到的一个命令

下载安装Node.js

Node.js官网

Hexo是依赖于Node.Js上的一个博客框架，Hexo 通过Node.Js渲染将Markdown转化成HTML文件，这样我们就可以直接将这些HTML文件推送到GitHub pages 上来托管我们的静态博客。

建议直接安装长期维护版，比较稳定。

一直按Next 就可以了（默认的就好）

打开cmd（命令提示符）窗口，输入node -v然后回车。

出现版本号便是安装成功了

Hexo官网

hexo官网

现在准备安装hexo

  1. 新建文件夹用于存放Hexo的文件（以后的博客也会存在这里面）

    打开新建Blog文件夹，右键使用“Open Git Bash here”命令打开窗口（红线的便是我们文件的地址位置）

  1. 打开Hexo官网

  1. 分别复制以上代码放在“Open Git Bash here”命令窗口中，每一行进行一次回车命令。
  1. 软件生成了本地的博客网址（红色方框的地址），在浏览器中打开即可以看见我们本地的博客。

  1. 基础本地化配置完成。

初始化目录以后是这个结构

目录结构说明

  • package.json: 这个是nodejs要用到的，里面存放着项目信息，版本，项目所依赖的第三方nodejs工具和代码库
  • package-lock.json: 这个nodejs解析依赖包时会用到的
  • scaffolds: 博客脚手架，当你创建新的博文时可以基于其中的某个模板进行修改．
  • source: 源代码文件夹，这里存放你网站的内容．
  • themes: 主题，博客css样式，背景图片等等．
  • node_modules: 这里存放npm install时从网络下载下来的依赖包
  • config.yml 项目配置文件

配置文件详细说明

  • 这里重点说明一下项目配置文件．
  • 你可以修改项目的默认配置文件_config.yml以满足你个人的需求．比如标题，网站的描述，搜索关键字等等一些信息
setting chines desc
title 标题 你网站的标题
subtitle 子标题 你网站的子标题
description 网站描述 你网站的描述信息
keywords 关键字 关键字
author 作者 网站的作者
language 语言 网站支持的语言

编辑个人博客

  1. 在Blog文件夹中右键打开“Open Git Bash here”（Git窗口），在窗口中输入hexo new "文章标题"回车即可得到新建的文章。文章储存位置为\Hexo\blog\source\_posts在“_posts”文件夹下面

文章为md格式，是用markdown语法进行编辑的（可以用typora软件进行书写）

  1. 保存文章后 在“Open Git Bash here”窗口（Git窗口）敲打命令行

  2. 运行hexo g加回车生成网页

  3. 运行hexo s加回车运行博客

  4. 在打开生成的本地博客网址，就可以发现博客已经更新了

Hexo主题使用

  1. 在Hexo主页下面有诸多主题样式可供选择，我们打开主题可进行挑选。

这里我应用的主题是”butterfly”，所以以这个主题样式进行讲解。

  1. 进入作者的GitHub主页，直接翻找中文的安装教程

这里直接附上链接Butterfly 安裝文檔(一) 快速開始 | Butterfly

网站是繁体中文的，但是在网站的右下角提供了中文简体字转化按钮，可以转化为中文简体。

大陆安装建议使用Gitee进行安装。

依旧是打开Git窗口将文档中提供的命令进行复制回车即可。

注意要安装文档中提供的插件才能正常运转啊。

  1. 安装完成后会在themes文件夹中生成butterfly文件夹

  2. 应用此主题要在blog文件夹中修改_config.yml文件。（建议先复制个副本，以免改坏了无法还原）

用记事本打开即可，将theme： 后面主题修改为butterfly（如果是其他主题，请按照刚刚themes文件夹中的名字进行修改）（注意：“theme:”后面要加空格“ ”再写上主题的名字）

  1. 保存文件修改后 在“Open Git Bash here”窗口（Git窗口）敲打命令行
  2. 运行hexo g加回车生成网页
  3. 运行hexo s加回车运行博客
  4. 出现报错就关闭Git窗口（可以直接Ctrl C）、浏览器窗口
